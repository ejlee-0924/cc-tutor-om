💡 WHY

6

📄
💡 WHY

"survey-sample.csv 전체 분석 — 총 응답 수, 평균 만족도, 항목별 평균, 주관식 긍정/부정 분류"

4
Subagent
병렬 작업 분배
🚀
💡 WHY
M2에서 파일을 하나씩 처리했는데, 교육 준비 시 코치일정/GPT계정/에그 등 독립적인 확인 작업이 많다. Subagent를 쓰면 여러 비서에게 각자 일을 맡기듯 동시에 처리하여 시간 대폭 단축.
🛠 실습 내용 — 4 Steps
1
병렬 처리 체험 — "박민수/이서연/김지훈 3명의 4월 빈 날짜를 동시에 찾아줘" → 서브에이전트 동시 실행
2
교육 준비 3종 병렬 — ① 코치 가용 여부 ② GPT 계정 가용 번호 ③ 에그 대여 가능 여부를 동시에 확인
3
종료/회고 데이터 병렬 분석 — review-log를 3가지 관점(강사별 만족도, 기업별 비교, 이슈 유형별 빈도)에서 동시 분석 → reports/ 저장
4
병렬 가능 여부 판별 — 4가지 작업 조합이 병렬 가능한지 직접 판별하는 실습
📌 주간 미션
실제 업무에서 동시에 처리할 수 있는 작업 2개를 찾아 Subagent로 실행. 순차 vs 병렬 소요 시간 차이를 #cc-tutor-om-help에 공유.

강의관리 — "이번 달 종료 과정 중 만족도 4.5 미만인 건을 찾아줘" (복합 조건 필터링)

5-3
Forms + Sheets + Gmail
만족도 E2E 파이프라인
💡 WHY
교육 종료 후 만족도 설문 → 응답 수집 → 분석 → 리포트 → 메일이 OM 핵심 업무. 현재 4개 앱을 오가며 수작업 100분+. MCP로 한 번에 파이프라인 처리.
🛠 실습 내용 — 5 Steps
1
"한국전자 AI특강 만족도 설문지를 Google Forms로 만들어줘" — 5점 척도 3개 + 주관식 1개
2
"설문 응답 결과를 확인하고 분석해줘" — 응답 조회 + 항목별 평균 + 주관식 긍정/부정 분류 한 번에
3
"강의 결과 리포트를 reports/에 저장해줘" — 과정명, 일시, 항목별 평균, 주관식 요약, 개선 제안
4
"강사님에게 만족도 결과 메일을 임시저장해줘" — draft_gmail_message (실제 발송 금지)
5
"싱크업 시트에서 담당자 이메일 조회 → 격식체로 교육 결과 메일 draft" — Sheets 조회 → Gmail 복합 작업
📌 주간 미션
실제 만족도 설문 1건을 Google Forms로 생성 (또는 기존 설문 응답 분석). 분석 결과를 리포트로 저장. 강사/담당자 메일을 draft로 작성. 결과를 #cc-tutor-om-help에 공유.

팀 전파 + GitHub
배포 · 온보딩 · Session Wrap
📦
💡 WHY
M1~M8에서 만든 CLAUDE.md, 스킬, AGENTS.md가 내 컴퓨터에만 있다. GitHub에 올리면 누구든 git clone 한 번으로 동일 환경. 신규 입사자도 클론 → 설치 → 바로 업무 투입.
🛠 실습 내용 — 4 Steps
1
"GitHub, repo, clone, commit, push를 OM 업무 비유로 설명해줘" — 기초 이해
2
레포 준비부터 Push까지 한 흐름 — README 정비 → .gitignore 확인 → git init → commit → push 방법 안내
3
/my-session-wrap 실행 — 이번 세션 작업 자동 정리 체험
4
신규 OM 온보딩 가이드 작성 — 7단계: git clone → CC 설치 → CLAUDE.md 확인 → MCP 설정 → 시트 접근 → /교안검수 실행 → 첫 업무 완료
📌 주간 미션
GitHub에 레포를 하나 만들어보기 (cc-tutor-om 또는 개인 프로젝트). git init → commit → push 과정 체험. README.md 작성. 결과를 #cc-tutor-om-help에 공유.

종합 실전 프로젝트 (Capstone)
M1~M9 통합 · 본인 업무 1건 E2E 자동화
🎖️
💡 WHY
M1~M9에서 10가지 핵심 기능을 모두 배웠다. 이제 본인의 실제 업무 하나를 처음부터 끝까지 자동화. Step1 Gems의 최종 진화 버전을 만든다.
🛠 실습 내용 — 6 Steps
1
주제 선정 — 예시: 매달 만족도 E2E 자동화, 분기별 교안 10개 일괄 검수, 강사 매칭 E2E 등
2
현재 상태 분석 — 몇 단계? 각 단계 소요 시간? 어디서 반복/병목?
3
E2E 설계 (Plan 모드) — 어떤 기능 조합? CLAUDE.md 추가 규칙? 스킬? AGENTS.md? MCP?
4
구현 — CLAUDE.md 규칙 추가 + 스킬 제작 + AGENTS.md 작성 + MCP 연동
5
테스트 — 실제 데이터(my-data/) 또는 samples/로 전체 파이프라인 실행 + 검증
6
팀 공유 — README.md 작성 + 사용법 가이드 + GitHub 올리기 (또는 준비)
📌 주간 미션
프로젝트를 완성하고 팀 사례 공유회에서 발표. Before/After 비교(시간, 품질, 반복 횟수). 활용한 Claude Code 기능 목록. 팀원이 따라 할 수 있는 재현 가이드.
