## 1. 배경 및 목표

### 현재 문제
- M2~M8에서 `slide-sample.pptx`, `survey-sample.csv` 반복 사용 (각각 4회, 5회)
- 실무 문서(Google Sheets)와의 연결 부재
- MCP 모듈(M5)에서 Slack/Notion/Gmail만 다루고, OM 핵심 도구인 Google Sheets/Forms/Calendar/Drive 미포함
- workspace-mcp, 세일즈맵 MCP 미포함

### 목표
1. M3부터 실무 문서 기반 실습으로 전환 (M1~M2는 기존 유지)
2. M5를 4개 서브모듈(5-1 ~ 5-4)로 확장하여 MCP를 충분히 다룸
3. M5 이후 모듈(M6~M8)이 MCP 결과물을 이어서 활용하는 누적 구조
4. 전체 모듈이 OM 업무 라이프사이클(인계→준비→운영→종료/분석→회고)과 연결

## 2. OM 업무 라이프사이클

수집한 실무 문서 8개를 분석한 결과, OM 업무는 교육 건 단위로 5단계 사이클을 반복한다.

### ① 인계 (LD → OM)
| 업무 | 문서 | 데이터 소스 |
|------|------|------------|
| 싱크업 시트 인수 | 싱크업 시트 | Sheets |
| 빈칸 보완 (불완전한 싱크업) | 세일즈맵 + Gmail + Slack | 세일즈맵/Gmail/Slack MCP |
| OM 배정 확인 | 위클리 시트 (OM 배정 탭) | Sheets |

### ② 준비 (교육 전)
| 업무 | 문서 | 데이터 소스 |
|------|------|------------|
| 강의관리 시트 생성 | 강의관리 시트 템플릿 | Sheets |
| 코치 일정 확인 + 배정 | 코치일정 캘린더 | Sheets |
| GPT 계정 배정 | GPT 계정 관리 시트 | Sheets |
| 에그 대여 예약 | 에그 관리 대장 | Sheets |
| 만족도 설문 제작 | Google Forms | Forms MCP |
| 체크리스트 진행 (25+ 항목) | 강의관리 시트 (체크리스트 탭) | Sheets |
| 강사/수강생 안내 메일 | 운영 템플릿 (메일/문자 링크) | Gmail MCP |

### ③ 운영 (교육 중)
| 업무 | 문서 |
|------|------|
| 현장 세팅 + OT | 강의관리 시트 (체크리스트) |
| 강의 요약 기록 | 강의관리 시트 (강의요약 탭) |
| 이슈 대응 + 기록 | 위클리 시트 |

### ④ 종료/분석 (교육 후)
| 업무 | 데이터 소스 |
|------|------------|
| 만족도 설문 응답 수집 | Forms + Sheets MCP |
| 만족도 분석 | Sheets MCP |
| 강의 결과 리포트 생성 | 로컬 |
| 강사/담당자에게 결과 메일 | Gmail MCP |
| LD에게 Slack 공유 | Slack MCP |
| GPT 계정 회수 (PW 변경 + 시트 업데이트) | Sheets |
| 에그 반납 확인 | Sheets |

### ⑤ 회고/공유
| 업무 | 데이터 소스 |
|------|------------|
| 종료/회고 시트 기록 | Sheets |
| 이슈/BP 패턴 분석 | Sheets |
| 위클리 미팅 공유 | Sheets/Slack |

## 3. 설계 원칙

### 접근법: 하이브리드 (확정)
- M1~M2: 기존 샘플 파일 유지 (기초 도구 학습)
- M3~M4: 실무 데이터를 CSV 샘플로 도입 (MCP 없이 로컬 파일)
- M5: MCP 집중 학습 (4개 서브모듈), 학습용 시트 사본 사용
- M6~M8: M5에서 배운 MCP를 스킬화/팀화하는 누적 구조
- M9~M10: 최소 변경 (주제 후보 확장)

### 데이터 전략

| 단계 | M1~M2 | M3~M4 | M5~ |
|------|-------|-------|-----|
| 데이터 | 기존 샘플 파일 | 신규 CSV 샘플 (가명 처리) | 학습용 시트 사본 (MCP 접근) |
| 읽기/쓰기 | 로컬 파일 | 로컬 파일 | 사본에 자유롭게 읽기+쓰기 |
| 원본 보호 | - | - | 원본 시트 절대 접근 안 함 |

### 학습용 시트 사본 (사전 준비 — 관리자 1회)

관리자(이은지 님)가 사전에 학습용 사본 1벌을 생성하고, 학습자 전원에게 편집 권한 부여.

| 학습용 사본 | 원본 |
|------------|------|
| [학습용] 코치일정 캘린더 | 코치일정 캘린더 |
| [학습용] GPT 계정 관리 | GPT 계정 관리 시트 |
| [학습용] 에그 관리 대장 | 에그 관리 대장 |
| [학습용] 위클리 시트 | 위클리 시트 (종료/회고) |
| [학습용] 강의관리 시트 | 강의관리 시트 템플릿 |
| [학습용] 싱크업 시트 | 싱크업 시트 |

CLAUDE.md에 학습용 시트 ID를 기록하여 학습자가 자동으로 올바른 시트에 접근하도록 한다.

### MCP 설치 범위 (학습자 직접 설치)

| MCP 서버 | 용도 |
|----------|------|
| workspace-mcp | Gmail, Sheets, Calendar, Drive, Docs, Forms |
| Slack MCP | 채널 읽기/쓰기 |
| 세일즈맵 MCP | 딜 정보 조회 |
