# 트러블슈팅 가이드

> 모듈별 자주 막히는 포인트와 해결 방법. 이은지 님 코칭 인사이트 기반.

## 공통 문제

### "Claude Code가 응답이 너무 느려요"
- 큰 파일을 읽고 있을 수 있음 → 조금 기다려주세요
- 너무 복잡한 요청은 나눠서 하세요 (이은지 님: "워크플로우 분해")

### "영어로 답변해요"
- "한국어로 답변해줘" 한 마디면 해결
- CLAUDE.md에 "항상 한국어로 응답" 규칙 추가하면 영구 해결

### "이전에 한 작업을 기억 못 해요"
- Claude Code는 같은 세션 내에서만 맥락 유지
- 중요한 결과는 파일로 저장해두면 나중에 다시 읽을 수 있음

### "뭘 요청해야 할지 모르겠어요"
- 자연어로 원하는 결과를 말하면 됨: "교안 검수해줘", "만족도 분석해줘"
- 구체적일수록 좋음: "이 PPT에서 오탈자 찾아줘" > "이거 확인해줘"

## M1: 작업환경 세팅 + CLAUDE.md

### "npm 명령어를 못 찾아요"
- Node.js가 설치되지 않은 것. https://nodejs.org 에서 LTS 설치
- 설치 후 터미널을 새로 열고 다시 시도

### "Claude Code 설치가 안 돼요"
- `npm install -g @anthropic-ai/claude-code` 실행
- 권한 오류 시: `sudo npm install -g @anthropic-ai/claude-code`
- 그래도 안 되면 Node.js 버전 확인: `node --version` (18 이상 필요)

### "API 키 설정은 어떻게?"
- 킥오프 세션에서 안내한 방법 따르기
- 잊어버렸으면 #cc-tutor-om-help에 문의

### "터미널이 무서워요"
- 괜찮습니다! Claude Code 안에서는 한국어로 대화하면 됩니다
- 코드를 직접 쓸 필요 없어요 — Claude Code가 씁니다

### "CLAUDE.md를 어디에 만들어야 하나요?"
- 프로젝트 최상위 폴더(루트)에 만듭니다
- Claude Code가 자동으로 인식

## M2: 파일 다루기 + Plan 모드

### "python-pptx 설치 에러 (PEP 668)"
- macOS에서 `pip install python-pptx` 시 차단될 수 있음
- 해결 방법 1: `pip install --user python-pptx`
- 해결 방법 2: `pip install --break-system-packages python-pptx`
- 해결 방법 3 (권장): 가상환경 생성 후 설치
  ```
  python -m venv .venv && source .venv/bin/activate && pip install python-pptx
  ```

### "PPT 읽었는데 텍스트가 안 나와요"
- 이미지 기반 슬라이드(텍스트가 이미지로 들어간 경우)는 텍스트 추출 불가
- 이 경우 PDF로 변환 후 읽기 시도

### "PPT 수정 후 디자인이 망가졌어요"
- python-pptx의 알려진 한계. 텍스트 교체만 하고 레이아웃은 건드리지 않도록 요청
- 복잡한 수정은 Claude in PowerPoint 사용 권장
- 이은지 님 코칭: "어설프게 고쳐진 PPT보다 완벽한 오답 노트가 현실적"

### "커리큘럼 PDF가 스캔 이미지예요"
- "이 PDF는 이미지 기반이니 OCR로 읽어줘" 요청
- 안 되면 텍스트를 직접 타이핑하여 curriculum.md로 만들기

### "CSV 한글 깨짐"
- "UTF-8로 다시 읽어줘" 또는 "EUC-KR 인코딩으로 읽어줘" 요청
- 엑셀에서 CSV 저장 시 "UTF-8 CSV" 선택

### "앱스크립트가 뭔지 모르겠어요"
- 구글 시트 → 확장 프로그램 → Apps Script
- 코드 편집기가 열리면 거기에 코드를 붙여넣고 실행 버튼 클릭

## M3: Skill 사용

### "스킬이 뭔지 감이 안 와요"
- /명령어를 입력하면 자동으로 복잡한 작업이 실행되는 것
- 예: /교안검수 → PPT 읽기 + 오류 분류 + 리포트 작성을 한 번에

### "설치한 스킬이 안 보여요"
- 설치 경로 확인: `~/.claude/skills/` (글로벌) 또는 `.claude/skills/` (프로젝트)
- Claude Code를 재시작하면 새 스킬 인식

### "기존 스킬(my-context-sync 등)은 어떻게 써요?"
- /my-context-sync 처럼 슬래시 + 스킬명으로 호출
- "어떤 스킬이 있는지 보여줘"라고 물어보면 목록 안내

## M4: Subagent

### "병렬 처리가 순차보다 느려요"
- 파일이 작으면 오버헤드 때문에 병렬이 더 느릴 수 있음
- 파일 3개 이상, 각 파일이 충분히 클 때 병렬이 유리

### "Subagent 결과가 하나만 돌아와요"
- 네트워크 불안정 시 일부만 완료될 수 있음
- "나머지 결과도 보여줘" 또는 다시 요청

### "스캔 이력서 PDF를 못 읽어요"
- OCR 시도: "이미지 기반 PDF니까 OCR로 읽어줘"
- 안 되면 이력서 핵심 내용을 텍스트로 정리하여 입력

## M5: MCP + Context Sync

### "MCP 서버 연결이 안 돼요"
- 설정 파일 경로 확인: `~/.claude/mcp.json`
- 토큰이 만료되었을 수 있음 → 새 토큰 발급 후 재설정
- "MCP 연결 상태 확인해줘"라고 요청

### "Slack 메시지 전송 권한 오류"
- Slack 앱 권한에 `chat:write` 스코프가 있는지 확인
- 채널에 앱이 초대되어 있는지 확인

### "Notion DB 접근이 안 돼요"
- Notion 통합(Integration)에 해당 데이터베이스 공유 여부 확인
- 토큰의 접근 권한 범위 확인

### "Context Sync 결과가 비어있어요"
- 연결된 MCP 서버 중 하나라도 오류면 전체가 비어 보일 수 있음
- 각 서비스를 하나씩 테스트하여 어디서 문제인지 확인

---

## M6: Plugin + Clarify

### "Clarify가 뭐에요? Plugin이랑 Skill 차이?"
- **Skill**: 명령어 한 줄로 실행되는 자동화 (ex: /교안검수)
- **Plugin**: 여러 스킬을 묶은 패키지 (ex: Clarify = vague + unknown + metamedium)
- Clarify는 모호한 요구사항을 구조화해주는 Plugin

### "Clarify 응답이 너무 길어요"
- Clarify는 체계적 질문을 3라운드로 진행함 — 원래 긴 편
- 빠른 결과가 필요하면 "핵심만 3줄로 정리해줘" 추가 요청

---

## M7: Skill Creator

### "스킬 만들기가 복잡해요"
- "이 워크플로우를 스킬로 만들어줘"라고 요청하면 Claude Code가 자동 생성
- 폴더 구조 + SKILL.md를 알아서 만들어줍니다

### "Eval 테스트가 뭔지 모르겠어요"
- "이 스킬이 제대로 작동하는지 테스트해줘"라고 요청
- 입력값을 바꿔가며 의도한 결과가 나오는지 확인하는 것

### "내가 만든 스킬이 안 돼요"
- SKILL.md 파일 위치 확인: `.claude/skills/{스킬명}/SKILL.md`
- SKILL.md 형식 오류 확인: frontmatter(---)가 올바른지 확인

## M8: Agent Teams

### "AGENTS.md를 어떻게 만들어요?"
- CLAUDE.md와 같은 위치(프로젝트 루트)에 생성
- 역할별 에이전트 정의: 검수 에이전트, 분석 에이전트, 메일 작성 에이전트 등

### "파이프라인 중간에 멈춰요"
- 특정 단계에서 오류 발생 → "어디서 멈췄는지 확인해줘" 요청
- 멈춘 단계만 다시 실행: "2단계(분석)부터 다시 해줘"

### "에이전트끼리 결과가 달라요"
- 의도된 것일 수 있음 — 다관점 검증의 핵심
- 차이가 큰 부분은 사람이 최종 판단

## M9: 팀 전파 + GitHub

### "GitHub 사용법을 모르겠어요"
- "GitHub에 올리는 방법 단계별로 알려줘" 요청
- 팀 내 GitHub 경험자와 피어 버디 매칭 추천

### "git push가 거부돼요"
- 원격 저장소에 변경이 있을 수 있음 → `git pull` 먼저 실행
- 권한 오류 → GitHub 토큰/SSH 키 확인

### "Session Wrap 결과가 너무 길어요"
- 자동 요약이 기본이므로 길어 보이면 "핵심만 요약해줘" 요청
- 결과 파일은 `logs/` 폴더에 자동 저장됨

## M10: 종합 실전 프로젝트

### "E2E 프로젝트 주제를 뭘로 잡아야 할까요?"
- 본인이 가장 반복적으로 하는 업무 1건을 선택
- 예: 매주 하는 만족도 정리 → Slack 보고, 분기별 강사 매칭 등

### "프로젝트가 너무 커요"
- 전체를 한 번에 하지 말고 단계별로 나누기
- 이은지 님 코칭: "프롬프트가 뚱뚱하면 오류 증가 — 일 나눠서 시키기"

## 이은지 님 코칭 인사이트 (참고)

1. **항상 Pro 모드 사용** — 빠른 모드에서 잘 안 나올 때 모드 설정이 문제
2. **AI 결과물 vs 직접 한 결과물 비교 검증 필수** — 처음엔 반드시 비교
3. **인풋 데이터 크기 테스트** — 점차 키우며 고장나는 지점 파악
4. **프롬프트가 뚱뚱하면 오류 증가** — 일 나눠서 시키기
5. **입력 형식별 성능**: 이미지 < PDF < 워드 순으로 성능 향상
6. **에이전트 분리**: 분석용 + 메일 작성용으로 나누면 성능 향상
7. **메타 학습**: AI 답변과 내 답변 비교 → "어떤 지시를 안 넣었길래?" 물어보기
