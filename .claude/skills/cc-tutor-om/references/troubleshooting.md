# 트러블슈팅 가이드

> 모듈별 자주 막히는 포인트와 해결 방법. 이은지 님 코칭 인사이트 기반.

## 공통 문제

### "Claude Code가 응답이 너무 느려요"
- 큰 파일을 읽고 있을 수 있음 → 조금 기다려주세요
- 너무 복잡한 요청은 나눠서 하세요 (이은지 님: "워크플로우 분해")

### "영어로 답변해요"
- "한국어로 답변해줘" 한 마디면 해결
- CLAUDE.md에 "항상 한국어로 응답" 규칙 추가하면 영구 해결

### "이전에 한 작업을 기억 못 해요"
- Claude Code는 같은 세션 내에서만 맥락 유지
- 중요한 결과는 파일로 저장해두면 나중에 다시 읽을 수 있음

### "뭘 요청해야 할지 모르겠어요"
- 자연어로 원하는 결과를 말하면 됨: "교안 검수해줘", "만족도 분석해줘"
- 구체적일수록 좋음: "이 PPT에서 오탈자 찾아줘" > "이거 확인해줘"

## 모듈 1: Claude Code 첫 만남

### "npm 명령어를 못 찾아요"
- Node.js가 설치되지 않은 것. https://nodejs.org 에서 LTS 설치
- 설치 후 터미널을 새로 열고 다시 시도

### "API 키 설정은 어떻게?"
- 킥오프 세션에서 안내한 방법 따르기
- 잊어버렸으면 #cc-tutor-om-help에 문의

### "터미널이 무서워요"
- 괜찮습니다! Claude Code 안에서는 한국어로 대화하면 됩니다
- 코드를 직접 쓸 필요 없어요 — Claude Code가 씁니다

## 모듈 2: 교안 검수 업그레이드

### "python-pptx 설치 에러"
- "pip install python-pptx 실행해줘" 요청
- 또는 "pip install --user python-pptx" 시도

### "PPT 읽었는데 텍스트가 안 나와요"
- 이미지 기반 슬라이드(텍스트가 이미지로 들어간 경우)는 텍스트 추출 불가
- 이 경우 PDF로 변환 후 읽기 시도

### "PPT 수정 후 디자인이 망가졌어요"
- python-pptx의 알려진 한계. 텍스트 교체만 하고 레이아웃은 건드리지 않도록 요청
- 복잡한 수정은 Claude in PowerPoint 사용 권장
- 이은지 님 코칭: "어설프게 고쳐진 PPT보다 완벽한 오답 노트가 현실적"

### "커리큘럼 PDF가 스캔 이미지예요"
- "이 PDF는 이미지 기반이니 OCR로 읽어줘" 요청
- 안 되면 텍스트를 직접 타이핑하여 curriculum.md로 만들기

## 모듈 3: 만족도 분석 파이프라인

### "CSV 한글 깨짐"
- "UTF-8로 다시 읽어줘" 또는 "EUC-KR 인코딩으로 읽어줘" 요청
- 엑셀에서 CSV 저장 시 "UTF-8 CSV" 선택

### "분석 결과가 이전 과정 데이터와 섞여요"
- 조여경 님 사례: 구글 폼 응답 시트에 작년 데이터가 포함
- "날짜 필터링해서 2026년 데이터만 분석해줘" 요청

### "메일 톤이 안 맞아요"
- 기존에 보낸 메일 예시를 보여주고 "이 톤으로 써줘" 요청
- 구체적 지시: "더 격식 있게" 또는 "더 친근하게"

### "파이프라인 스크립트 실행이 안 돼요"
- "이 스크립트의 오류를 찾아 수정해줘" 요청 (모듈 4 디버깅 기법 활용)
- Python이 설치되어 있는지 확인: `python --version`

## 모듈 4: 앱스크립트 디버깅

### "앱스크립트가 뭔지 모르겠어요"
- 구글 시트 → 확장 프로그램 → Apps Script
- 코드 편집기가 열리면 거기에 코드를 붙여넣고 실행 버튼 클릭

### "권한 오류"
- 첫 실행 시 권한 승인 팝업 → "허용" 클릭
- "이 앱은 확인되지 않았습니다" → "고급" → "프로젝트로 이동"

### "수정한 코드가 기존 데이터를 망가뜨릴까 걱정"
- 시트 복사본에서 먼저 테스트
- "테스트용으로 원본 데이터를 건드리지 않는 코드로 만들어줘" 요청

## 모듈 5: 강사 매칭 자동화

### "스캔 이력서 PDF를 못 읽어요"
- OCR 시도: "이미지 기반 PDF니까 OCR로 읽어줘"
- 안 되면 이력서 핵심 내용을 텍스트로 정리하여 입력

### "매칭 점수가 모든 후보에게 비슷하게 나와요"
- 커리큘럼의 역량 기준이 너무 포괄적일 수 있음
- "필수 역량에만 가중치를 두고 다시 매칭해줘" 요청

## 이은지 님 코칭 인사이트 (참고)

1. **항상 Pro 모드 사용** — 빠른 모드에서 잘 안 나올 때 모드 설정이 문제
2. **AI 결과물 vs 직접 한 결과물 비교 검증 필수** — 처음엔 반드시 비교
3. **인풋 데이터 크기 테스트** — 점차 키우며 고장나는 지점 파악
4. **프롬프트가 뚱뚱하면 오류 증가** — 일 나눠서 시키기
5. **입력 형식별 성능**: 이미지 < PDF < 워드 순으로 성능 향상
6. **에이전트 분리**: 분석용 + 메일 작성용으로 나누면 성능 향상
7. **메타 학습**: AI 답변과 내 답변 비교 → "어떤 지시를 안 넣었길래?" 물어보기
