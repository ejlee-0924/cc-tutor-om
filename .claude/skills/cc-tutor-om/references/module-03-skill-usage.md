# 모듈 3: Skill 사용

> 배우는 기능: 스킬 탐색, 설치 (npx skills add), 실행 (/명령어), 스킬 생태계 이해
> OM 실습: /교안검수 스킬 체험 + 기존 글로벌/회사 스킬 탐색 + 외부 스킬 설치

---

## WHY

M2에서 PPT 검수하고 CSV 분석하는 과정을 매번 처음부터 지시했다.
"PPT 읽어줘 → 커리큘럼 대비 검수해줘 → 프로토콜 적용해줘 → 리포트 만들어줘" — 이걸 매번?

스킬을 쓰면 `/교안검수` 한 마디면 끝난다.

**Before (M2까지):**
- 매번 5단계 지시를 반복
- 지시 누락 시 결과 품질 차이
- 팀원에게 전달하려면 절차 설명 필요

**After (Skill 사용):**
- `/교안검수` 한 마디 → 파일 찾기 → 분석 → 리포트 자동 생성
- 누가 실행해도 동일 품질
- 팀원도 같은 스킬 설치하면 바로 사용

**공식 문서:** https://docs.anthropic.com/en/docs/claude-code/skills

## EXPLAIN

### Skill = 원클릭 매크로

| 비유 | 일반 지시 | 스킬 |
|------|---------|------|
| 요리 | 매번 레시피 읽으며 요리 | "김치찌개" 버튼 → 자동 조리 |
| OM 업무 | "PPT 읽고, 검수하고, 리포트 써줘" | `/교안검수` |

### 스킬의 3가지 레벨

| 레벨 | 위치 | 범위 | 예시 |
|------|------|------|------|
| **프로젝트 스킬** | `.claude/skills/` | 이 프로젝트에서만 | /교안검수 |
| **글로벌 스킬** | `~/.claude/skills/` | 모든 프로젝트에서 | /my-context-sync |
| **외부 스킬** | 커뮤니티 | 설치 후 사용 | npx skills add로 설치 |

### 스킬 vs 직접 지시

| | 직접 지시 | 스킬 |
|---|---|---|
| 재현 가능성 | 낮음 (지시 표현 따라 다름) | 높음 (동일 결과) |
| 품질 일관성 | 들쭉날쭉 | 일정 |
| 팀 공유 | 절차 설명 필요 | 스킬 파일 공유만으로 끝 |
| 커스터마이징 | 매번 수정 | SKILL.md 수정 후 영구 반영 |

## EXECUTE

**Step 1 — /교안검수 스킬 체험**

    /교안검수

(미리 만들어진 스킬이 실행되면) 자동으로:
- samples/ 에서 PPT + 커리큘럼 PDF를 찾고
- 검수 프로토콜 적용하여 분석하고
- 리포트를 생성하는 과정을 관찰한다.

M2에서 5단계로 지시했던 것이 한 마디로 끝나는 것을 체감한다.

**Step 2 — 기존 스킬 목록 확인**

    지금 사용 가능한 스킬 목록을 보여줘

설치된 글로벌/회사/프로젝트 스킬 목록을 확인한다.

**Step 3 — 글로벌 스킬 체험: /my-context-sync**

    /my-context-sync

(설치되어 있는 경우) Slack, Notion, Gmail 등에서 정보를 수집하여 한 문서로 정리하는 과정을 체험한다.

**Step 4 — 글로벌 스킬 체험: /my-content-fetch**

    아무 URL을 제공하고 /my-content-fetch로 콘텐츠를 가져와 요약하는 것을 체험한다.

**Step 5 — 외부 스킬 설치**

    npx skills add 명령어를 통해 커뮤니티에서 유용한 스킬을 검색하고 설치한다.

**Step 6 — 스킬 내부 구조 살펴보기**

    .claude/skills/cc-tutor-om/SKILL.md를 읽어줘.
    스킬이 어떤 구조로 만들어졌는지 설명해줘.

M7(Skill Creator)에서 직접 만들 때를 위한 사전 이해.

## QUIZ

**Q1.** 스킬과 직접 지시의 가장 큰 차이는?

- A) 스킬이 더 느리다
- B) 스킬은 누가 실행해도 동일한 결과를 보장한다
- C) 직접 지시가 더 정확하다
- D) 차이가 없다

**정답: B**
> 스킬은 표준화된 워크플로우입니다. 직접 지시는 표현에 따라 결과가 달라질 수 있지만, 스킬은 동일한 절차를 보장합니다.

**Q2.** 글로벌 스킬과 프로젝트 스킬의 차이는?

- A) 글로벌은 모든 프로젝트에서, 프로젝트 스킬은 이 프로젝트에서만 동작
- B) 글로벌이 더 강력하다
- C) 프로젝트 스킬은 팀원과 공유할 수 없다
- D) 차이가 없다

**정답: A**
> 글로벌 스킬(`~/.claude/skills/`)은 어디서든 사용 가능, 프로젝트 스킬(`.claude/skills/`)은 해당 프로젝트에서만 동작합니다. CLAUDE.md의 프로젝트/개인 레벨과 같은 원리입니다.

## TROUBLESHOOTING

### "스킬이 실행 안 돼요"
- 스킬 이름 정확한지 확인: `/교안검수` (슬래시 포함)
- 스킬 파일이 `.claude/skills/` 폴더에 있는지 확인

### "npx skills add가 안 돼요"
- Node.js가 설치되어 있는지 확인
- 네트워크 연결 확인
- `npm cache clean --force` 후 재시도

### "어떤 스킬이 있는지 모르겠어요"
- "사용 가능한 스킬 목록 보여줘" 요청
- 또는 `.claude/skills/` 폴더 직접 확인

### "스킬 결과가 기대와 다라요"
- SKILL.md를 읽어서 스킬이 어떤 절차로 동작하는지 확인
- 입력 데이터가 스킬이 기대하는 형식인지 확인
