# 모듈 5-2: Google Sheets 조회 + 입력

> 배우는 기능: Sheets 읽기(read_sheet_values), 쓰기(modify_sheet_values), 시트 정보(get_spreadsheet_info)
> OM 실습: 코치일정/GPT계정/에그/종료회고 시트 조회 + 입력

---

## WHY

OM 업무의 70%가 Google Sheets다.
코치일정 확인, GPT 계정 배정, 에그 대여 기록, 종료/회고 작성 — 매일 4~5개 시트를 오가며 작업한다.
Claude Code에서 바로 처리하면 시트 전환 시간을 절약한다.

**Before (수동):**
- 시트 열기 → 탭 이동 → 필터 → 데이터 확인 → 다른 시트 열기 → 반복
- 시트 4~5개를 왔다 갔다 하면서 정보 수집
- 수작업 입력 시 오타, 빈칸 실수

**After (MCP Sheets):**
- "이번 달 코치일정 빈 날짜 찾아줘" → 바로 답변
- "GPT 계정 1~15번 배정 기록해줘" → 한 번에 입력
- "미반납 에그 목록 확인해줘" → 즉시 조회

## EXPLAIN

### Sheets 도구 3가지

| 도구 | 기능 | 사용 예 |
|------|------|---------|
| `read_sheet_values` | 시트 데이터 읽기 (범위 지정 가능) | 코치일정 조회, 만족도 데이터 읽기 |
| `modify_sheet_values` | 특정 셀/범위 수정 | 일정 추가, 계정 배정 기록 |
| `get_spreadsheet_info` | 시트 메타 정보 (탭 목록, 크기) | 어떤 탭이 있는지 확인 |

### 읽기 vs 쓰기 흐름

```
읽기: read_sheet_values → 데이터 확인 → 분석/답변
쓰기: read_sheet_values(현재 상태 확인) → modify_sheet_values(수정) → read_sheet_values(결과 확인)
```

쓰기 전에 항상 읽기로 현재 상태를 먼저 확인하는 것이 안전하다.

### 학습용 시트 사본

- 원본 시트를 직접 수정하면 위험 → 학습용 사본에서 자유롭게 연습
- 학습용 시트는 CLAUDE.md에 등록되어 있어 Claude Code가 자동으로 인식

### 동시 편집 주의

여러 명이 같은 시트에서 동시에 실습하면 충돌할 수 있다.

| 규칙 | 예시 |
|------|------|
| 각자 다른 날짜/행에 입력 | 나는 4/15, 동료는 4/16 |
| 이니셜+날짜로 구분 | JY_0424, MH_0424 |
| 같은 셀 동시 수정 금지 | 작업 전 시트 확인 |

## EXECUTE

---

### Step 1 — 코치일정 조회

📋 아래를 복사해서 입력하세요:

    [학습용] 코치일정 캘린더 시트에서 박민수 코치의 이번 달 빈 날짜를 찾아줘.

read_sheet_values로 코치일정 시트를 읽고, 빈 날짜를 필터링하여 보여준다.

---

### Step 2 — 코치일정 입력

📋 아래를 복사해서 입력하세요:

    [학습용] 코치일정 캘린더 시트에서 박민수 코치 4/15에 "한국전자 AI특강"을 추가해줘.

modify_sheet_values로 해당 셀에 데이터를 입력한다. 입력 후 결과를 다시 읽어서 확인.

---

### Step 3 — GPT 계정 조회

📋 아래를 복사해서 입력하세요:

    [학습용] GPT 계정 관리 시트에서 4/15 사용 가능한 GPT 계정 번호를 알려줘.

사용 중이 아닌 계정 번호를 필터링하여 목록으로 보여준다.

---

### Step 4 — GPT 계정 입력

📋 아래를 복사해서 입력하세요:

    [학습용] GPT 계정 관리 시트에서 1~15번 계정을 한국전자 AI특강 과정에 배정 기록해줘. 날짜는 4/15.

여러 셀을 한 번에 수정한다. 범위 지정 쓰기의 예시.

---

### Step 5 — 종료/회고 조회

📋 아래를 복사해서 입력하세요:

    [학습용] 강의관리 시트에서 이번 달 종료 과정 중 만족도 4.5 미만인 건을 찾아줘.

조건부 필터링: 종료 상태 + 만족도 점수 기준으로 해당 건을 추출한다.

## QUIZ

**Q1.** Sheets 읽기(read_sheet_values)와 쓰기(modify_sheet_values)의 올바른 사용 순서는?

- A) 바로 쓰기부터 한다
- B) 읽기로 현재 상태 확인 → 쓰기 → 읽기로 결과 확인
- C) 쓰기만 하면 된다
- D) 읽기만 반복한다

**정답: B**
> 쓰기 전에 현재 데이터를 확인하고, 쓰기 후에 결과를 검증하는 것이 안전한 패턴입니다. 실수로 잘못된 셀을 덮어쓰는 것을 방지합니다.

**Q2.** 학습용 시트 사본을 쓰는 이유는?

- A) 원본보다 빠르기 때문에
- B) 원본 데이터를 보호하면서 자유롭게 연습하기 위해
- C) MCP가 원본을 읽을 수 없기 때문에
- D) 팀장이 시켜서

**정답: B**
> 학습 중에 실수로 원본 데이터를 수정하거나 삭제하면 실제 업무에 영향을 줍니다. 학습용 사본에서 마음껏 연습하세요.

## TROUBLESHOOTING

### "시트에 접근이 안 돼요"
- 학습용 시트의 공유 권한 확인 (편집자 권한 필요)
- workspace-mcp가 연결되어 있는지 `/mcp`로 확인
- 시트 ID가 CLAUDE.md에 올바르게 등록되어 있는지 확인

### "범위 지정이 안 먹어요"
- 시트명에 한글이 포함된 경우: 작은따옴표로 감싸기 (예: '코치일정'!A1:D10)
- 탭 이름이 정확한지 `get_spreadsheet_info`로 먼저 확인
- 범위 형식: A1:D10 (시작셀:끝셀)

### "입력했는데 다른 셀에 들어갔어요"
- 쓰기 전에 `read_sheet_values`로 현재 상태를 먼저 확인
- 셀 주소(행/열)가 정확한지 재확인
- 입력 후 반드시 다시 읽어서 결과 검증

### "한글 시트명이 인식이 안 돼요"
- 시트 탭 이름을 정확히 입력 (공백, 특수문자 주의)
- `get_spreadsheet_info`로 탭 목록을 먼저 조회하여 정확한 이름 확인
