# 모듈 10: 종합 실전 프로젝트

> 배우는 기능: M1~M9 전체 기능 통합 활용
> OM 실습: 본인의 실제 업무 1건을 E2E 자동화 — 환경설정→파일처리→스킬화→팀 공유

---

## WHY

M1~M9에서 Claude Code의 10가지 핵심 기능을 모두 배웠다.
이제 배운 것을 종합하여 본인의 실제 업무 하나를 처음부터 끝까지 자동화한다.

Step1에서 만든 에이전트의 최종 진화 버전을 만든다.

**Before (M1 시작 전):**
- Gems로 단순 질답만 가능
- 파일 처리 불가, 환각 위험, 단일 작업만

**After (M10 완료):**
- CLAUDE.md로 규칙 자동 적용
- 파일 직접 처리 + Plan 모드로 계획적 실행
- 스킬로 원클릭 실행
- Subagent로 병렬 처리
- MCP로 Slack/Notion 연동
- Agent Teams로 파이프라인 자동화
- GitHub으로 팀 공유

## EXPLAIN

### 캡스톤 프로젝트 = 졸업 작품

M1~M9에서 배운 기능을 하나의 프로젝트에 통합한다.

### 프로젝트 설계 프레임워크

```
1. 주제 선정: 어떤 반복 업무를 자동화할 것인가?
2. 현재 상태 분석: 지금은 어떻게 하고 있는가? (시간, 단계, 반복)
3. 설계 (Plan 모드): 어떤 기능을 조합할 것인가?
4. 구현: CLAUDE.md + 스킬 + AGENTS.md + MCP
5. 테스트: 실제 데이터로 검증
6. 공유: GitHub + 온보딩 가이드
```

### 추천 프로젝트 주제 (OM 업무)

| 주제 | 활용 기능 | Step1 연결 |
|------|---------|-----------|
| **월간 교육 결과 보고서 자동화** | 파일(M2) + 스킬(M7) + Teams(M8) + Slack(M5) | 조여경·이혜림 |
| **교안 일괄 검수 시스템** | 파일(M2) + Subagent(M4) + 스킬(M7) | 김세린·박규은·임지민·박현정 |
| **강사 매칭 파이프라인** | 파일(M2) + Subagent(M4) + Teams(M8) | 홍정이·안서연 |
| **교육 기획 요구사항 자동 구조화** | Clarify(M6) + 스킬(M7) | 전원 |
| **신규 OM 온보딩 키트** | CLAUDE.md(M1) + GitHub(M9) | 전원 |

## EXECUTE

**Step 1 — 프로젝트 주제 선정**

    M1~M9에서 배운 기능 중, 본인의 업무에 가장 적용하고 싶은 것은?
    반복적으로 하는 업무 중 자동화하고 싶은 것 1개를 선택하세요.

    예시:
    - 매달 하는 만족도 분석 → 보고서 → 메일 전체 자동화
    - 분기별 교안 10개 일괄 검수
    - 강사 매칭 프로세스 E2E 자동화

**Step 2 — 현재 상태 분석**

    선택한 업무를 지금은 어떻게 하고 있는지 정리해줘:
    - 몇 단계인가?
    - 각 단계에 얼마나 걸리는가?
    - 어디서 반복/병목이 발생하는가?

**Step 3 — E2E 설계 (Plan 모드)**

    이 업무를 자동화하는 전체 계획을 세워줘. Plan 모드로:

    - 어떤 Claude Code 기능을 조합할 것인가?
    - CLAUDE.md에 추가할 규칙은?
    - 스킬로 만들 워크플로우는?
    - AGENTS.md가 필요한가?
    - MCP 연동이 필요한가?

**Step 4 — 구현**

계획에 따라 단계별로 구현한다:
1. CLAUDE.md에 프로젝트 규칙 추가
2. 핵심 워크플로우를 스킬로 제작
3. 필요 시 AGENTS.md 작성 (TF 파이프라인)
4. 필요 시 MCP 연동 설정

**Step 5 — 테스트**

    실제 데이터(my-data/)로 전체 파이프라인을 실행해줘.
    결과를 검증해줘.

(my-data/에 데이터가 없으면 samples/로 테스트)

**Step 6 — 팀 공유**

    이 프로젝트를 팀원이 바로 사용할 수 있도록:
    1. README.md 작성
    2. 사용법 가이드 작성
    3. GitHub에 올리기 (또는 올릴 준비)

## TROUBLESHOOTING

### "어떤 주제를 선택해야 할지 모르겠어요"
- 지난 한 달간 가장 시간을 많이 쓴 반복 업무는?
- "내 업무 중 자동화하면 좋을 것을 추천해줘" 요청
- 추천 주제 목록(EXPLAIN 섹션) 참고

### "너무 복잡해서 어디서부터 시작해야 할지 모르겠어요"
- Plan 모드로 전체 계획을 먼저 세우기
- 가장 쉬운 단계부터 하나씩 구현
- 완벽하지 않아도 됨 — "동작하는 v1"이 먼저

### "실제 데이터가 없어요"
- samples/ 데이터로 먼저 프로토타입
- 프로토타입이 동작하면 실제 데이터로 전환
- 데이터 익명화 방법: "이 데이터의 이름을 가명으로 바꿔줘"

### "팀원이 관심이 없어요"
- 먼저 본인이 사용하며 효과 입증
- Before/After 데이터로 구체적 성과 보여주기
- "한 번 써봐" 보다 "이거 쓰면 월 5시간 아낄 수 있어" 가 효과적
