# 모듈 5-3: Forms + Sheets + Gmail (만족도 E2E)

> 배우는 기능: Google Forms 생성(create_form), 응답 조회(list_form_responses), Gmail 임시저장(draft_gmail_message)
> OM 실습: 만족도 설문 생성 → 응답 수집 → 분석 → 리포트 → 강사/담당자 메일 임시저장

---

## WHY

교육 종료 후 만족도 설문 → 응답 수집 → 분석 → 리포트 → 메일이 OM 핵심 업무다.
현재는 Forms 열고, Sheets 열고, 분석하고, Gmail 열고... 4개 앱을 오가며 수작업.
MCP로 한 번에 처리한다.

**Before (수작업):**
- Forms 제작 (30분) → 응답 수집 → Sheets 분석 (20분) → 리포트 작성 (30분) → 메일 3통 (20분) = 100분+
- 앱 4개를 왔다 갔다, 복사-붙여넣기 반복

**After (MCP E2E):**
- "만족도 설문 만들고, 분석하고, 결과 메일 초안 만들어줘" = 한 번에
- Forms 생성 → 응답 분석 → 리포트 → 메일 임시저장까지 파이프라인

⚠️ **안전 규칙:** 학습 모드에서는 `draft_gmail_message`(임시저장)만 사용. 실제 발송(`send_gmail_message`) 금지.

---

## EXPLAIN

### E2E 파이프라인 구조

```
Forms 생성 → 응답 수집 → Sheets 분석 → 리포트 저장 → Gmail 임시저장
(create_form) (list_form_responses) (read_sheet_values) (파일 저장) (draft_gmail_message)
```

### 주요 도구

| 도구 | 기능 | 사용 시점 |
|------|------|----------|
| `create_form` | Google Forms 설문지 생성 | 만족도 설문 제작 |
| `list_form_responses` | 설문 응답 목록 조회 | 응답 수집 후 |
| `get_form_response` | 개별 응답 상세 조회 | 특정 응답 확인 |
| `read_sheet_values` | 응답 데이터 분석용 읽기 | 통계 처리 |
| `draft_gmail_message` | 메일 임시보관함에 저장 | 결과 메일 작성 |

### draft vs send 차이

| 구분 | draft_gmail_message | send_gmail_message |
|------|--------------------|--------------------|
| 동작 | 임시보관함에 저장만 | 실제 발송 |
| 취소 가능 | ✅ 발송 전 수정/삭제 가능 | ❌ 발송 후 취소 불가 |
| 학습 모드 | ✅ 사용 가능 | ❌ 사용 금지 |
| 용도 | 메일 초안 작성 → 확인 후 수동 발송 | 실제 업무용 (학습 후) |

### 운영 템플릿 활용

mail-templates-sample.csv에 정의된 메일 형식(만족도 결과 전달, 강사 안내 등)을 활용하여 일관된 메일 초안 작성.

## EXECUTE

---

### Step 1 — 만족도 설문 생성

📋 아래를 복사해서 입력하세요:

    한국전자 AI특강 만족도 설문지를 Google Forms로 만들어줘.
    항목:
    - 강의내용 만족도 (5점 척도)
    - 강사역량 만족도 (5점 척도)
    - 실습만족도 (5점 척도)
    - 개선사항 (주관식)

create_form으로 설문지가 생성되고, 설문 URL이 반환된다.

---

### Step 2 — 응답 분석

📋 아래를 복사해서 입력하세요:

    설문 응답 결과를 확인하고 분석해줘:
    - 총 응답 수, 항목별 평균 점수
    - 주관식 응답을 긍정/부정으로 분류
    - 전체 요약 코멘트

list_form_responses로 응답을 조회하고, 항목별 통계와 주관식 분류 결과를 정리한다.

---

### Step 3 — 결과 리포트 생성

📋 아래를 복사해서 입력하세요:

    강의 결과 리포트를 만들어서 reports/ 폴더에 저장해줘.
    포함 내용: 과정명, 일시, 항목별 평균, 주관식 요약, 개선 제안.

분석 결과를 정리하여 마크다운 리포트로 저장한다.

---

### Step 4 — 강사 메일 임시저장

📋 아래를 복사해서 입력하세요:

    강사 김지훈님에게 만족도 결과 메일을 임시저장해줘.
    운영 템플릿의 '만족도 결과 전달' 형식으로 작성하고,
    항목별 평균과 주관식 요약을 포함해줘.

draft_gmail_message로 임시보관함에 저장된다. Gmail에서 확인 후 수동 발송.

---

### Step 5 — 기업 담당자 메일 임시저장

📋 아래를 복사해서 입력하세요:

    [학습용] 싱크업 시트에서 한국전자 담당자 이메일을 조회하고,
    격식체로 교육 결과 메일을 임시저장해줘.
    항목별 평균, 수강생 반응 요약, 향후 제안 포함.

Sheets 조회(담당자 이메일) → Gmail 임시저장까지 연결하는 복합 작업.

## QUIZ

**Q1.** Forms → Sheets → Gmail 파이프라인에서 MCP가 하는 역할은?

- A) 각 앱을 자동으로 설치해준다
- B) Forms 생성, 응답 조회, Sheets 분석, Gmail 임시저장을 하나의 흐름으로 연결해준다
- C) Google 계정을 자동으로 만들어준다
- D) 설문 응답자에게 자동으로 알림을 보낸다

**정답: B**
> MCP는 각 Google Workspace 서비스를 Claude Code에서 직접 호출할 수 있게 연결합니다. 덕분에 앱을 오가지 않고 하나의 흐름으로 처리할 수 있습니다.

**Q2.** 학습 모드에서 draft_gmail_message를 쓰는 이유는?

- A) send보다 빠르기 때문에
- B) 임시보관함에 저장만 하므로 실수로 발송되는 것을 방지할 수 있기 때문에
- C) draft가 더 예쁘게 보이기 때문에
- D) send 기능이 없기 때문에

**정답: B**
> draft는 임시보관함에 저장만 합니다. 메일 내용을 직접 확인한 후 수동으로 발송하므로, 학습 중 실수로 잘못된 메일이 나가는 것을 방지합니다.

---

## LEADER-TIP

1. **E2E 파이프라인 데모**: Forms→분석→메일을 한 번에 보여주면 팀원의 체감 효과가 크다
2. **메일 품질 검수**: draft로 만든 메일을 발송 전 반드시 검수하는 습관 강조
3. **운영 템플릿 표준화**: 팀 공용 메일 템플릿을 정리하여 일관된 커뮤니케이션 유지
4. **안전 규칙 반복 강조**: send는 실무 전환 후에만. 학습 중에는 항상 draft

## TROUBLESHOOTING

### "Forms 생성이 안 돼요"
- workspace-mcp가 연결되어 있는지 `/mcp`로 확인
- Google 계정에 Forms 생성 권한이 있는지 확인
- 질문 항목 형식이 올바른지 확인 (척도, 주관식 등)

### "응답 데이터가 없어요"
- 설문에 실제 응답이 입력되었는지 확인
- 학습용 더미 응답이 없다면 직접 2~3개 테스트 응답 입력
- Form ID가 올바른지 확인

### "Gmail draft 오류가 나요"
- workspace-mcp의 Gmail 인증이 완료되었는지 확인
- 수신자 이메일 형식이 올바른지 확인 (예: user@company.com)
- 메일 본문이 너무 길면 분할 요청

### "메일 형식이 깨져요"
- 운영 템플릿 형식을 명시적으로 지정: "한글 격식체로, 표 포함"
- HTML 형식 대신 텍스트 형식으로 요청
- 줄바꿈이 필요하면 "각 항목을 줄바꿈으로 구분해줘" 지시
