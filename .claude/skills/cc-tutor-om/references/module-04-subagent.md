# 모듈 4: Subagent

> 배우는 기능: 병렬 작업 분배, 독립 태스크 판별, 서브에이전트 활용
> OM 실습: 코치 3명 일정 동시 조회, 교육 준비 3종 병렬 확인, 종료/회고 다관점 병렬 분석

---

## WHY

M2에서 파일을 하나씩 처리하는 법을 배웠다. 그런데 교육 준비할 때 확인할 게 한두 개가 아니다.
코치일정 확인, GPT 계정 가용 확인, 에그 대여 가능 여부 — 각각 시트를 열어서 하나씩 확인하면 시간이 너무 걸린다.

Subagent를 쓰면 여러 비서에게 각자 일을 맡기듯, 독립적인 작업을 동시에 처리한다.

**Before (순차 처리):**
- 코치일정 확인 → GPT 계정 확인 → 에그 확인 → 각각 시트 열어서 확인 → 15분
- 종료/회고 분석 → 강사별 → 기업별 → 이슈별 → 순서대로 → 대기 시간

**After (Subagent 병렬):**
- 코치일정 + GPT 계정 + 에그 동시 확인 → 3분
- 강사별 + 기업별 + 이슈별 동시 분석 → 절반 시간

**공식 문서:** https://docs.anthropic.com/en/docs/claude-code/sub-agents

## EXPLAIN

### Subagent = 여러 비서에게 동시에 일 맡기기

| 방식 | 비유 | 상황 |
|------|------|------|
| 순차 처리 | 비서 1명이 모든 일 순서대로 | 코치일정 확인 → GPT 계정 → 에그 |
| 병렬 처리 (Subagent) | 비서 3명이 각자 동시에 | 비서A: 코치일정, 비서B: GPT 계정, 비서C: 에그 |

### 병렬 가능 vs 불가능 판별법

| 상황 | 병렬 가능? | 이유 |
|------|-----------|------|
| 코치 3명 일정 각각 조회 | ✅ 가능 | 서로 독립적 |
| 코치일정 + GPT 계정 + 에그 동시 확인 | ✅ 가능 | 서로 독립적 |
| 만족도 분석 → 분석 결과로 메일 작성 | ❌ 불가능 | 메일이 분석 결과에 의존 |
| 종료/회고 데이터 강사별/기업별 동시 분석 | ✅ 가능 | 서로 독립적 |

**핵심 질문**: "앞 작업 결과가 다음 작업에 필요한가?"
- 아니오 → 병렬 가능
- 예 → 순차 처리

### Agent 도구 활용

Claude Code에서 서브에이전트는 `Agent` 도구로 실행된다.
자연어로 "이 3개를 동시에 해줘"라고 요청하면 Claude Code가 알아서 병렬 분배한다.

## EXECUTE

---

### Step 1 — 병렬 처리 체험: 코치 3명 동시 조회

📋 아래를 복사해서 입력하세요:

    coach-schedule-sample.csv에서 다음 코치 3명의 4월 빈 날짜를 동시에 찾아줘:
    1. 박민수
    2. 이서연
    3. 김지훈

    각각 별도로 결과를 정리해줘.

세 작업이 서브에이전트로 동시에 실행되는 것을 확인한다.

---

### Step 2 — 교육 준비 3종 병렬 확인

📋 아래를 복사해서 입력하세요:

    다음 3가지를 동시에 확인해줘:
    1. coach-schedule-sample.csv에서 김지훈 코치 4/15 가용 여부
    2. gpt-accounts-sample.csv에서 4/15 사용 가능한 계정 번호
    3. egg-sample.csv에서 4/15 대여 가능한 에그

    각각 결과를 정리해줘.

코치일정, GPT 계정, 에그 — 서로 독립적인 3가지가 동시에 확인되는 것을 관찰한다.

---

### Step 3 — 종료/회고 데이터 병렬 분석

📋 아래를 복사해서 입력하세요:

    review-log-sample.csv를 다음 3가지 관점에서 동시에 분석해줘:
    1. 강사별 평균 만족도 순위
    2. 기업별 만족도 비교
    3. 이슈 유형별 빈도 분석 (현장운영/수강생/강의내용/기타)

    각 분석을 reports/ 폴더에 별도 저장해줘.

세 가지 분석이 병렬로 진행되고, 각각 리포트가 생성되는 것을 확인한다.

---

### Step 4 — 병렬 가능 여부 판별 실습

📋 아래를 복사해서 입력하세요:

    다음 작업들이 병렬 가능한지 판별해줘:
    1. 코치일정 확인 + GPT 계정 확인
    2. 만족도 분석 → 분석 결과로 메일 작성
    3. 에그 A 대여 기록 + 에그 B 대여 기록
    4. 싱크업 시트 읽기 → 빈칸 기반 체크리스트 생성

## QUIZ

**Q1.** Subagent를 쓰면 좋은 상황은?

- A) 모든 상황에서 항상 써야 한다
- B) 서로 독립적인 작업이 여러 개일 때
- C) 순서가 중요한 작업일 때
- D) 파일이 1개일 때

**정답: B**
> Subagent는 "독립적인 작업의 동시 처리"입니다. 서로 결과를 주고받아야 하는 작업은 순차 처리가 맞습니다.

**Q2.** "만족도 분석 → 분석 결과로 메일 작성"은 병렬 처리가 가능한가?

- A) 가능하다
- B) 불가능하다 — 메일이 분석 결과에 의존하므로 순차 처리해야 한다
- C) 절반만 가능하다
- D) Subagent로만 가능하다

**정답: B**
> 핵심 질문: "앞 작업 결과가 다음 작업에 필요한가?" 메일 작성은 분석 결과가 있어야 하므로 순차 처리입니다.

## LEADER-TIP

1. **교육 준비 병렬화**: "코치일정 + GPT 계정 + 에그 동시 확인" — 준비 시간 단축
2. **종료/회고 다관점 분석**: 강사별/기업별/이슈별을 동시에 분석하여 인사이트 발굴
3. **시간 절약 체감**: 순차 vs 병렬 소요 시간을 직접 비교하여 팀에 공유
4. **주의사항 전파**: "다 병렬로 하면 안 된다 — 의존 관계 있는 건 순차"

## TROUBLESHOOTING

### "병렬 처리가 안 되고 순서대로 되요"
- Claude Code가 작업 간 의존성을 판단하여 자동으로 순차 처리할 수 있음
- "이 두 작업은 독립적이니 동시에 해줘"라고 명시적으로 요청

### "서브에이전트 결과가 서로 달라요"
- 각 서브에이전트는 독립적으로 실행되므로, CLAUDE.md 규칙이 각각 적용됨
- 결과 형식을 맞추려면 "모든 리포트는 동일한 양식으로" 지시

### "너무 많은 파일을 동시에 하면 느려져요"
- 한 번에 3-5개 정도가 적당
- 너무 많으면 나눠서 진행: "먼저 5개, 완료 후 나머지 5개"

### "코치 이름이 CSV에 없어요"
- coach-schedule-sample.csv의 코치명 컬럼 확인
- 정확한 이름(박민수, 이서연, 김지훈)으로 요청
