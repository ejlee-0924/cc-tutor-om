# 모듈 4: Subagent

> 배우는 기능: 병렬 작업 분배, 독립 태스크 판별, 서브에이전트 활용
> OM 실습: PPT 3개 동시 검수, 검수+만족도 분석 병렬 실행, 이력서 일괄 분석

---

## WHY

M2에서 파일을 하나씩 처리하는 법을 배웠다. 그런데 교안 PPT가 10개면? 만족도 설문이 5건이면?
하나씩 순서대로 하면 시간이 너무 걸린다.

Subagent를 쓰면 여러 비서에게 각자 일을 맡기듯, 독립적인 작업을 동시에 처리한다.
박현정 님의 600페이지도, 이혜림 님의 대량 분석도 한 번에.

**Before (순차 처리):**
- PPT 1 검수 → PPT 2 검수 → PPT 3 검수 → 30분
- 만족도 분석 → 교안 검수 → 순서대로 → 대기 시간

**After (Subagent 병렬):**
- PPT 1, 2, 3 동시 검수 → 10분
- 만족도 분석 + 교안 검수 동시 → 절반 시간

**공식 문서:** https://docs.anthropic.com/en/docs/claude-code/sub-agents

---

## EXPLAIN

### Subagent = 여러 비서에게 동시에 일 맡기기

| 방식 | 비유 | 상황 |
|------|------|------|
| 순차 처리 | 비서 1명이 모든 일 순서대로 | PPT 검수 → 분석 → 메일 |
| 병렬 처리 (Subagent) | 비서 3명이 각자 동시에 | 비서A: PPT, 비서B: CSV, 비서C: 메일 |

### 병렬 가능 vs 불가능 판별법

| 상황 | 병렬 가능? | 이유 |
|------|-----------|------|
| PPT 3개 각각 검수 | ✅ 가능 | 서로 독립적 |
| 검수 + 만족도 분석 동시 | ✅ 가능 | 서로 독립적 |
| 분석 → 분석 결과로 메일 작성 | ❌ 불가능 | 메일이 분석 결과에 의존 |
| 이력서 3개 각각 분석 | ✅ 가능 | 서로 독립적 |

**핵심 질문**: "앞 작업 결과가 다음 작업에 필요한가?"
- 아니오 → 병렬 가능
- 예 → 순차 처리

### Agent 도구 활용

Claude Code에서 서브에이전트는 `Agent` 도구로 실행된다.
자연어로 "이 3개를 동시에 해줘"라고 요청하면 Claude Code가 알아서 병렬 분배한다.

## EXECUTE

---

### Step 1 — 병렬 vs 순차 개념 이해

먼저 순차 처리를 체험한다:

📋 아래를 복사해서 입력하세요:

    slide-sample.pptx를 검수해줘.

(완료 후)

📋 아래를 복사해서 입력하세요:

    survey-sample.csv를 분석해줘.

두 작업이 순서대로 실행되는 것을 확인한다.

---

### Step 2 — 병렬 처리 체험

📋 아래를 복사해서 입력하세요:

    다음 두 가지를 동시에 해줘:
    1. slide-sample.pptx를 검수 프로토콜로 검수
    2. survey-sample.csv를 긍정/부정으로 분류 분석

    각각 별도의 리포트로 저장해줘.

두 작업이 서브에이전트로 동시에 실행되는 것을 확인한다.

---

### Step 3 — 대량 파일 병렬 처리

📋 아래를 복사해서 입력하세요:

    samples/ 폴더의 모든 파일을 각각 분석해줘:
    - PPT → 검수 리포트
    - CSV → 만족도 분석
    - JS → 버그 분석
    - PDF → 내용 요약

    각 파일별 결과를 reports/ 폴더에 저장해줘.

여러 서브에이전트가 각 파일을 동시에 처리하는 것을 관찰한다.

---

### Step 4 — 이력서 일괄 분석 (매칭 시나리오)

📋 아래를 복사해서 입력하세요:

    resume-sample.pdf의 3명 후보자를 curriculum-sample.pdf 기준으로 각각 매칭 분석해줘.
    후보자별로 별도 리포트를 만들고, 마지막에 비교 요약표도 만들어줘.

후보자별 분석이 병렬로 진행되고, 최종 비교는 순차로 합쳐지는 패턴을 확인한다.

---

### Step 5 — 병렬 가능 여부 판별 실습

📋 아래를 복사해서 입력하세요:

    다음 작업들이 병렬 가능한지 판별해줘:
    1. PPT 검수 + CSV 분석
    2. CSV 분석 → 분석 결과로 메일 작성
    3. 이력서 A 분석 + 이력서 B 분석
    4. 교안 수정 → 수정된 교안 검수

## QUIZ

**Q1.** Subagent를 쓰면 좋은 상황은?

- A) 모든 상황에서 항상 써야 한다
- B) 서로 독립적인 작업이 여러 개일 때
- C) 순서가 중요한 작업일 때
- D) 파일이 1개일 때

**정답: B**
> Subagent는 "독립적인 작업의 동시 처리"입니다. 서로 결과를 주고받아야 하는 작업은 순차 처리가 맞습니다.

**Q2.** "만족도 분석 → 분석 결과로 메일 작성"은 병렬 처리가 가능한가?

- A) 가능하다
- B) 불가능하다 — 메일이 분석 결과에 의존하므로 순차 처리해야 한다
- C) 절반만 가능하다
- D) Subagent로만 가능하다

**정답: B**
> 핵심 질문: "앞 작업 결과가 다음 작업에 필요한가?" 메일 작성은 분석 결과가 있어야 하므로 순차 처리입니다.

## LEADER-TIP

1. **대량 처리 시나리오 발굴**: "분기별 만족도 설문 5건 동시 분석", "교안 10개 일괄 검수"
2. **이은지 코칭 연결**: "워크플로우를 분해하면 병렬화 가능한 부분이 보인다"
3. **시간 절약 체감**: 순차 vs 병렬 소요 시간을 직접 비교하여 팀에 공유
4. **주의사항 전파**: "다 병렬로 하면 안 된다 — 의존 관계 있는 건 순차"

## TROUBLESHOOTING

### "병렬 처리가 안 되고 순서대로 되요"
- Claude Code가 작업 간 의존성을 판단하여 자동으로 순차 처리할 수 있음
- "이 두 작업은 독립적이니 동시에 해줘"라고 명시적으로 요청

### "서브에이전트 결과가 서로 달라요"
- 각 서브에이전트는 독립적으로 실행되므로, CLAUDE.md 규칙이 각각 적용됨
- 결과 형식을 맞추려면 "모든 리포트는 동일한 양식으로" 지시

### "너무 많은 파일을 동시에 하면 느려져요"
- 한 번에 3-5개 정도가 적당
- 너무 많으면 나눠서 진행: "먼저 5개, 완료 후 나머지 5개"

### "이력서가 여러 장인데 한 명만 분석돼요"
- "각 후보자별로 나눠서 분석해줘" 명시
- PDF에서 페이지별로 후보자가 구분되는지 확인
