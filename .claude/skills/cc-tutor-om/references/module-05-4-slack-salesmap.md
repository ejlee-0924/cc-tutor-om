# 모듈 5-4: Slack + 세일즈맵 + Context Sync

> 배우는 기능: Slack 읽기/쓰기, 세일즈맵 딜 조회(salesmap_search_records, salesmap_get_record), /my-context-sync
> OM 실습: 싱크업 빈칸 보완 + LD 결과 공유 + Context Sync 통합 수집

---

## WHY

LD(기업교육컨설턴트)가 싱크업 시트를 대충 써서 넘기면 OM이 힘들다.
빈칸을 채우려면 세일즈맵에서 딜 정보 찾고, Gmail에서 메일 뒤지고, Slack에서 대화 찾고...
이 3개를 합치면 싱크업이 자동으로 완성된다.

**Before (수동):**
- 싱크업 빈칸 발견 → 세일즈맵 로그인 → 딜 검색 → Gmail 검색 → Slack 검색 → 수동 입력 = 30분+
- 3개 앱을 오가며 정보 조각 수집

**After (MCP 통합):**
- "싱크업 빈칸 채워줘" → 세일즈맵 + Gmail + Slack에서 자동 수집 + 보완안 제시 = 5분
- Context Sync로 여러 소스를 한 문서로 통합

---

## EXPLAIN

### Slack MCP 도구

| 도구 | 기능 | OM 사용 예 |
|------|------|-----------|
| `slack_read_channel` | 채널 메시지 읽기 | 교육운영 채널 최근 소식 확인 |
| `slack_search_public` | 공개 채널 검색 | 특정 교육 건 관련 대화 찾기 |
| `slack_send_message` | 메시지 보내기 | 결과 보고, 이슈 공유 |

### 세일즈맵 MCP 도구

| 도구 | 기능 | OM 사용 예 |
|------|------|-----------|
| `salesmap_search_records` | 기업명/키워드로 검색 | "한국전자" 딜 찾기 |
| `salesmap_get_record` | 특정 레코드 상세 조회 | 딜 상세 정보 (금액, 담당자, 일정) |

⚠️ **세일즈맵 제한:** 시퀀스 조회는 MCP로 불가. 딜/레코드 조회는 가능.
시퀀스 정보가 필요하면 세일즈맵 웹에서 직접 확인.

### Context Sync = 멀티소스 통합 수집

```
/my-context-sync
  ├── Slack → 관련 채널 메시지 수집
  ├── Gmail → 관련 메일 수집
  ├── Sheets → 관련 시트 데이터 수집
  └── 통합 문서로 정리
```

여러 MCP 소스를 한 번에 수집하여 하나의 통합 문서로 만든다.
흩어진 정보를 모아 전체 그림을 파악할 때 유용하다.

## EXECUTE

---

### Step 1 — Slack 읽기

📋 아래를 복사해서 입력하세요:

    교육운영 채널의 최근 메시지를 읽어줘.

slack_read_channel로 지정 채널의 최근 메시지를 조회한다.

---

### Step 2 — 세일즈맵 딜 조회

📋 아래를 복사해서 입력하세요:

    한국전자 AI특강 딜 정보를 세일즈맵에서 찾아줘.
    금액, 담당 LD, 교육 일정, 특이사항을 알려줘.

salesmap_search_records로 기업명 검색 → salesmap_get_record로 상세 조회.

---

### Step 3 — 싱크업 빈칸 보완

📋 아래를 복사해서 입력하세요:

    [학습용] 싱크업 시트를 읽고 빈 필드를 찾아줘.
    세일즈맵, Gmail, Slack에서 정보를 수집해서 보완안을 제시해줘.

이 단계가 M5-4의 핵심이다. 여러 MCP 소스를 조합하여 빈칸을 채우는 워크플로우:
1. Sheets에서 싱크업 빈칸 식별
2. 세일즈맵에서 딜 정보 수집
3. Gmail에서 관련 메일 검색
4. Slack에서 관련 대화 검색
5. 수집된 정보로 보완안 제시

---

### Step 4 — 결과 Slack 공유

📋 아래를 복사해서 입력하세요:

    만족도 분석 결과 요약을 교육운영 Slack 채널에 공유해줘.
    항목별 평균과 핵심 인사이트를 포함해줘.

slack_send_message로 분석 결과를 채널에 직접 공유한다.

---

### Step 5 — /my-context-sync 체험

📋 아래를 복사해서 입력하세요:

    /my-context-sync를 실행해서 이번 주 교육 관련 정보를
    Slack, Gmail, Sheets에서 한 문서로 정리해줘.

Context Sync 스킬이 여러 소스를 병렬 수집하여 통합 문서를 생성한다.

## QUIZ

**Q1.** 싱크업 빈칸을 보완할 때 어떤 MCP 조합을 사용하나?

- A) Sheets만 사용
- B) Sheets(빈칸 확인) + 세일즈맵(딜 정보) + Gmail(메일) + Slack(대화)
- C) 세일즈맵만 사용
- D) Slack만 사용

**정답: B**
> 싱크업 빈칸은 여러 소스에 흩어진 정보를 모아야 채울 수 있습니다. Sheets로 빈칸을 확인하고, 세일즈맵/Gmail/Slack에서 필요한 정보를 수집하는 멀티소스 조합이 핵심입니다.

**Q2.** Context Sync의 가장 큰 장점은?

- A) 특정 앱 하나를 깊이 분석한다
- B) 여러 소스(Slack, Gmail, Sheets)를 한 번에 수집하여 통합 문서로 정리한다
- C) 자동으로 메일을 발송한다
- D) 세일즈맵 시퀀스를 조회한다

**정답: B**
> Context Sync는 흩어진 정보를 한 곳에 모아 전체 그림을 파악하게 해줍니다. 각 앱을 따로 열어 확인하는 시간을 절약합니다.

## LEADER-TIP

1. **싱크업 보완 시나리오**: 실제 빈칸이 많은 싱크업 건을 예시로 보여주면 체감 효과가 크다
2. **Slack 공유 습관화**: 분석 결과를 Slack에 바로 공유하면 팀 커뮤니케이션이 빨라진다
3. **세일즈맵 한계 사전 안내**: 시퀀스 조회 불가를 미리 알려서 혼란 방지
4. **Context Sync 주간 루틴**: 매주 월요일 Context Sync로 한 주 교육 건 전체 파악

## TROUBLESHOOTING

### "세일즈맵 검색 결과가 없어요"
- 기업명 철자 확인 (정식 명칭 vs 약칭)
- 검색어를 줄여서 재시도: "한국전자" 대신 "한국전자" 또는 "전자"
- 딜이 아직 등록되지 않았을 수 있음 → LD에게 확인

### "Slack 채널에 접근이 안 돼요"
- 해당 채널에 본인이 참여(join)되어 있는지 확인
- 비공개 채널은 slack_search_public으로 검색 불가 → slack_search_public_and_private 사용
- Slack MCP 연결 상태 `/mcp`로 확인

### "Context Sync 실행 오류"
- 3개 MCP(workspace-mcp, Slack, 세일즈맵)가 모두 연결되어 있는지 확인
- 검색 범위를 좁혀서 재시도: "이번 주" 대신 "오늘" 또는 특정 교육 건 지정
- 네트워크 상태 확인

### "세일즈맵 시퀀스 조회가 안 돼요"
- MCP로 시퀀스 조회는 현재 미지원
- 대안: 세일즈맵 웹에서 직접 확인하거나, 딜 레코드의 메모/노트에서 관련 정보 검색
- salesmap_get_record로 딜 상세를 조회하면 일부 정보는 확인 가능
