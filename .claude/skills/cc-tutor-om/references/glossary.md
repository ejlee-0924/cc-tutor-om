# OM 용어 사전 (비유 포함)

> 기술 용어를 OM 업무 맥락으로 풀어서 설명합니다.

## Claude Code 기본 (M1)

| 용어 | OM 비유 | 설명 |
|------|---------|------|
| **Claude Code** | 현장 출장 전문가 | 내 컴퓨터에서 직접 파일을 읽고 수정하는 AI |
| **Gems/GPTs** | 전화 상담 | 브라우저 안에서만 동작하는 AI |
| **터미널** | 명령어 입력 창 | 텍스트로 컴퓨터에 지시하는 창 |
| **CLI** | 텍스트 기반 인터페이스 | GUI(마우스 클릭)의 반대 개념 |
| **CLAUDE.md** | AI가 읽는 팀 매뉴얼 | 팀 규칙/기준을 AI가 자동으로 따르게 하는 파일 |
| **권한 모드** | 결재 수준 설정 | Claude Code가 파일을 수정/실행할 때 허용 범위 |

## Plugin + Clarify (M6)

| 용어 | OM 비유 | 설명 |
|------|---------|------|
| **Plugin** | 선택형 부품 | 필요할 때 끼워 쓰는 기능 확장 모듈 |
| **Clarify** | 기획 전 체크리스트 | 모호한 요구사항을 명확하게 만드는 3가지 모드 |
| **vague 모드** | "뭘 원하세요?" | 막연한 요청을 구체적으로 정리 |
| **unknown 모드** | "놓친 게 없나?" | 알려진/모르는 것을 4분면으로 분류 |
| **metamedium 모드** | "형식을 바꿔볼까?" | 내용 vs 형식 중 어디가 문제인지 재구성 |

## GitHub + 배포 (M9)

| 용어 | OM 비유 | 설명 |
|------|---------|------|
| **GitHub** | 팀 공유 드라이브 (버전 관리) | 코드/문서를 저장하고 버전을 관리하는 서비스 |
| **repo (레포)** | 프로젝트 폴더 | GitHub에 올린 프로젝트 전체 |
| **클론(clone)** | 폴더 복사해오기 | GitHub에서 내 컴퓨터로 프로젝트를 가져오는 것 |
| **커밋(commit)** | 저장 + 메모 | 변경사항을 기록하는 단위 |
| **푸시(push)** | 공유 드라이브에 업로드 | 내 컴퓨터의 커밋을 GitHub에 올리는 것 |
| **Session Wrap** | 퇴근 전 일지 | 세션 종료 시 작업 내역을 자동 정리하는 기능 |
| **Compound** | 고급 세션 관리 | 여러 세션을 연결하여 대규모 작업을 이어가는 기능 |
| **Hook** | 자동 트리거 | 특정 이벤트 발생 시 자동으로 실행되는 작업 |

## 검수 프로토콜 (박규은 모델)

| 용어 | 의미 |
|------|------|
| **PASS** | 수정 불필요 (Fatal 0건) |
| **CONDITIONAL** | 경미한 수정 (Fatal 0건, Quality 1~3건) |
| **REJECT** | 즉시 수정 필수 (Fatal 1건 이상) |
| **Fatal Error** | 치명적 오류 (내용 오류, 기업명 오타, 누락 섹션) |
| **Quality Issue** | 품질 개선점 (톤앤매너, 여백, 줄간격) |
